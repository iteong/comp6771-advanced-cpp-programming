Topics Covered:

Consider the following code: 
#include <iostream>

class A1 {
public:
  A1() { 
    std::cout << "A1 "; 
  }
};

class A2 {
public:
  A2() {
    std::cout << "A2 ";
    a1 = new A1();
  }
private:
  A1 *a1;
};

class A3 {
public:
  A3() {
    std::cout << "A3 ";
    a1 = new A1();
    a2 = new A2();
  }
private:
  A1 *a1;
  A2 *a2;
};

int main() {
  A3 *a3 = new A3();
}
  1. Work out on paper/whiteboard what the output of this program is.
  2. Change the declaration of class A3 to: 
    class A3  : public A2 {
    Work out what the new output is.
  3. Why are the outputs different?
Consider the following code: 
#include <iostream>

struct X { 
  ~X() { std::cout << "~X() "; }
};

struct Y { 
  ~Y() { std::cout << "~Y() "; }
};

class A { 
  X x;
public:
  ~A() { std::cout << "~A() "; }
};

class B : public A { 
  Y y;
public:
  ~B() { std::cout << "~B() "; }
};

int main() { 
  B b; 
}
  1. On paper/whiteboard work out the output of this program.
  2. Change the object B in main to a B* 
    B* b = new B;
    What is the output now? Why is the output different?
Consider the following code: 
#include <list>
#include <iostream>

class X {
public:
  X() { std::cout << "ctor "; }
  X(const X&) { std::cout << "copy-ctor "; }
  ~X() { std::cout << "dtor "; }
};

int main() {
  std::list<X*> l;
  X *x = new X();
  X y;
  l.push_back(x);
  l.push_back(&y);
}
  1. On paper/whiteboard work out the output of this program.
  2. Add the follow to the end of main: 
      
std::list<X> l2;
l2.push_back(y);
    Why is the copy constructor now called (and wasn't earlier), why is there an extra dtor print?
